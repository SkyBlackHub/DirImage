DirImage
Плагин для Total Commander 5.51 или выше, добавляет возможность быстрого просмотра директорий с изображениями в виде превью.
Работает как в режиме "просмотра эскизов", так и через внутренний просмотрщик файлов.

DirImage распространяется по лицензии GNU Lesser General Public License:
https://www.gnu.org/licenses/lgpl.html

Страница плагина:
https://github.com/SkyBlackHub/DirImage

Принципы работы:
Плагин работает только с папками. При вызове он сканирует целевую папку (опционально проверяя вложенные папки), ища в ней файлы имеющие допустимые расширения (см. extensions) и являющиеся изображениями.
Найденные в папке изображения отображаются постранично в виде таблицы эскизов с размером columns x rows.

Вся настройка осуществляется через файл DirImage.ini в папке плагина.
Стуктура INI-файла:
[general]
Основные настройки плагина.

language
Язык меню. Доступные языки: en - английский, ru - русский.

log
Уровень логирования, полезен при отладке. Доступные уровни:
none - выключить логирование
error - выводить только сообщения об ошибках
info - также сообщения информационного характера
debug - все сообщения
Значение по-умолчанию: none

log_file
Файл для вывода сообщений. Если не указан, сообщения будут выводиться в стандартные std::cerr и std::cout.

[view]
[thumbs]
[common]
Данные секции содержат основные настройки показа. Секция [view] отвечает за работу в режиме просмотрщика, а секция [thumbs] за работы в режиме просмотра эскизов.
Секция [common] может содержать общие для этих двух секций настройки.
Например, если задать rows=3 в [common], а в других секциях не задавать эту настройку, то для них будет использовано значение 3.
Если же скажем в секции [view] добавить rows=2, то для неё будет использовано именно это значение.

enabled
Включить/выключить работу в данном режиме.
Допустимые значения: true, false
Значение по-умолчанию: true

extensions
Список расширений файлов, указанные через пробел, которые будут использованы для превью.
На данный момент допустимы все типы файлов (включая популярные jpg, png, gif, bmp, tga и psd), что поддерживаются библиотекой FreeImage.
Полный список можно найти по адресу: http://freeimage.sourceforge.net/features.html и в документации: http://freeimage.sourceforge.net/documentation.html
Значение по-умолчанию: jpg jpeg gif png bmp gif webp

columns
Количество показываемых в превью изображений по горизонтали.
Значение по-умолчанию: 2

rows
Количество показываемых в превью изображений по вертикали.
Значение по-умолчанию: 2

background
Цвет фона в превью, задаётся тремя цифрами через пробел в формате RGB (0 0 0 - 255 255 255).
Значение по-умолчанию: 0 0 0

enlarge
Увеличивать или нет изображения, размер которых меньше текущего размера одной клетки в превью.
Текеущий размер клетки равен размеру окна/эскиза разделённому на количество изображений в нём (т.е. rows x columns).
Иными словами ширина клетки = ширина окна / columns, а высота клетки = высота окна / rows.
Допустимые значения: true, false
Значение по-умолчанию: false

ignore_dots
Игнорировать псевдокаталоги "..".
Удобно, если нужно быстро посмотреть информацию о каталоге с изображениями (т. е. не использовать DirImage).
Допустимые значения: true, false
Значение по-умолчанию: true

filter
Фильтр, используемый при масштабировании изображений.
На данный момент допустимы все фильтры, что поддерживаются библиотекой FreeImage. Подробнее об этих фильтрах можно почитать в документации по адресу: http://freeimage.sourceforge.net/documentation.html
Фильтр box наиболее быстрый и менее качественный из всех. Однако при уменьшении изображения он даёт вполне приемлимое качество, т. ч. если enlarge выключен, то именно этот фильтр является рекомендуемым.
Допустимые значения: box, bicubic, bilinear, bspline, catmullrom, lanczos3
Значение по-умолчанию: box

adaptive
Адаптивный размер сетки. Если количество изображений на странице меньше, чем площадь сетки, то её размер будет уменьшен, чтобы избежать пустот.
Значение по-умолчанию: true

transparency_grid
Показывать шахматную сетку в качестве фона у полупрозрачных изображений.
Значение по-умолчанию: true

pad_h
pad_v
Расстояние в пикселях между эскизами в сетке по горизонтали и вертикали
Значение по-умолчанию: 0, 0

shift
Пропускать указанное количество эскизов от начала при отображении.
Удобно, когда, например, не нужно показывать первое изображение.
Значение по-умолчанию: 0

files_limit
Максимальное количество проверенных файлов, после которых поиск в каталоге прекращается.
Полезно, чтобы предотвращать зависания на каталогах со слишком большим количеством файлов.
Значение по-умолчанию: 1000

deep_scan
Включить загрузку из подкаталогов, если в основном каталоге ничего не найдено.
Значение по-умолчанию: false

deep_scan_level
Уровень вложенности подкаталогов, на который можно опускаться в поисках изображений.
Значение по-умолчанию: 1

deep_scan_limit
Максимальное количество изображений, которое можно загружать из каждого подкаталога.
Значение по-умолчанию: 1

deep_scan_files_limit
То же самое, что files_limit, только для подкаталогов.
Имеет смысл использовать небольшие значения, чтобы избежать зависаний на объёмных по содержанию каталогах.
Значение по-умолчанию: 100

info
Показывать информационное сообщение (текущая страница, текущий диапазон изображений, общее количество и т.д.)
Значение по-умолчанию: 1

info_color
Цвет текста информационного сообщения, задаётся тремя цифрами через пробел в формате RGB (0 0 0 - 255 255 255).
Значение по-умолчанию: 0 255 0

info_size
Размер текста информационного сообщения.
Значение по-умолчанию: 16
